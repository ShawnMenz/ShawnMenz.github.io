安装Git后

$ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com"

创建repository

将一个目录变成Git可以管理的仓库

$ git init

把文件添加到仓库

$ git add file.txt

将文件提交到仓库

$ git commit -m "info"

版本控制

掌握仓库当前的状态（包括在哪一个分支，更改发生状况）

$ git status

查看difference，file.txt的更改情况

$ git diff file.txt

commit类似于“快照”，就像游戏中的存档，可以从某一个commit处恢复回滚。查看commit记录

$ git log

回滚

用HEAD表示当前版本，也就是最新的提交，上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100

$ git reset --hard HEAD^

前滚

需要找到commit id，然后

$ git reset --hard id

可以借助以下命令来查找commit id，它记录了用户输入的每一次命令

$ git reflog

撤销修改

可以根据git status的提示来进行命令输入

工作区撤销

让file.txt文件回到最近一次git commit或git add时的状态

$ git checkout -- file.txt

暂存区撤销

git reset命令既可以回退版本，也可以把暂存区的修改回退到工作区。当我们用HEAD时，表示最新的版本。

$ git reset HEAD file.txt

删除文件

在文件管理器中把文件删除之后会造成工作区和版本库不一致，如果该文件是误删，那么进行版本回退，如果确实需要删除

$ git rm file.txt

并且进行git commit

远程仓库

创建SSH Key

$ ssh-keygen -t rsa -C "youremail@example.com"

可以在.ssh目录里面找到id_rsa和id_rsa.pub。id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。

同一台电脑生成多个SSH Key

有时会出现一台电脑需要连接多个Git账户的情况，此时需要在一台电脑上面生成多个公钥和私钥并映射多个github账号

生成密钥文件

$ ssh-keygen -t rsa -C "name.github.com" -f ~/.ssh/id_rsa_test

通过ssh-add添加密钥至ssh-agent

$ ssh-add ~/.ssh/id_rsa_test

添加远程仓库

添加后，远程库的名字就是origin

$ git remote add origin git@github.com:xxx/xxx

把本地库的内容推送到远程，用git push命令，实际上是把当前分支master推送到远程。

$ git push -u origin master

-u参数会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

$ git push origin master

删除远程库

先用以下命令查看远程库信息

$ git remote -v

然后，根据名字删除，比如删除origin

$ git remote rm origin

此处的“删除”其实是解除了本地和远程的绑定关系，并不是物理上删除了远程库。远程库本身并没有任何改动。要真正删除远程库，需要登录到GitHub，在后台页面找到删除按钮再删除。

从远程库克隆

$ git clone git@github.com:xxx/xxx

分支管理

创建分支

创建dev分支并切换到上面，git checkout命令加上-b参数表示创建并切换

$ git checkout -b dev

新版的Git也能使用

$ git switch -c dev

用git branch命令查看当前分支

$ git branch

换到某一个分支，例如换回master

$ git checkout master

新版的Git也能使用

$ git switch master

合并分支

把dev分支的工作成果合并到master分支上

$ git merge dev

删除分支

删除dev分支

$ git branch -d dev

强行删除分支

$ git branch -D dev

工作现场

暂存工作现场

$ git stash

去别的分支修复完bug之后，恢复工作现场，并删除

$ git stash pop

或者分步完成

$ git stash apply
$ git stash drop

查看所有工作现场

$ git stash list
  • 在master分支上修复的bug，想要合并到当前dev分支，可以用 git cherry-pick <commit>命令，把bug提交的修改“复制”到当前分支，避免重复劳动。
  • git rebase操作可以把本地未push的分叉提交历史整理成直线
